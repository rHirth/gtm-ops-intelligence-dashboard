\# GTM Operations Intelligence Dashboard



This project is a synthetic GTM Operations analytics case study. It is designed to demonstrate how Python, SQL, dashboarding, and AI-assisted analysis can be used to diagnose operational friction across a go-to-market workflow.



\## Current Scope



Version 0.1 begins with synthetic global account generation. Future versions will add leads, opportunities, ticket triage, SLA tracking, SQL analysis, and an interactive dashboard.



\## Business Questions



The final dashboard will focus on three RevOps pain points:



1\. Leads that do not convert

2\. Opportunities that close lost

3\. Tickets that miss SLA



\## Synthetic Data Design



The data is synthetic and intentionally designed to simulate realistic GTM Operations patterns, including global territories, account segmentation, CRM data quality issues, routing challenges, and downstream operational risk.



The purpose of the project is not to make empirical claims about real companies. The purpose is to demonstrate data modeling, Python generation, SQL analysis, dashboard design, and executive communication.
